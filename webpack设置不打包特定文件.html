# webpack设置不打包某些文件

最近遇到了这种需求，要实现webpack打包后，一个全局配置文件config.js不被打包。最终目的是我们可以在打包后的文件中修改这个文件（比如配置全局请求url前缀等等）。

要实现这个功能我们需要三步操作。

  1. 了解webpack的整个大致打包流程
  2. 通过修改webpack配置文件，将某个文件拷贝到输出资源目录(而不是通过打包输出)
  3. 在模板index.html中引入

# webpack大致打包流程

通过查看网上的资料，可以了解到下面的打包流程:

1. 初始化参数：从配置文件和 Shell 语句中读取与合并参数，得出最终的参数；

2. 开始编译：用上一步得到的参数初始化 Compiler 对象，加载所有配置的插件，执行对象的 run 方法开始执行编译；

3. 确定入口：根据配置中的 entry 找出所有的入口文件；

4. 编译模块：从入口文件出发，调用所有配置的 Loader 对模块进行翻译，再找出该模块依赖的模块，再递归本步骤直到所有入口依赖的文件都经过了本步骤的处理；

5. 完成模块编译：在经过第4步使用 Loader 翻译完所有模块后，得到了每个模块被翻译后的最终内容以及它们之间的依赖关系；

6. 输出资源：根据入口和模块之间的依赖关系，组装成一个个包含多个模块的 Chunk，再把每个 Chunk 转换成一个单独的文件加入到输出列表，这步是可以修改输出内容的最后机会；

7. 输出完成：在确定好输出内容后，根据配置确定输出的路径和文件名，把文件内容写入到文件系统。

注： 参考自https://juejin.im/post/5d70b186f265da03e1689864

通过第4步我们可以得知，我们不能在任何会经过打包的文件中引入我们的config.js。因为webpack会从入口文件开始递归所有引入的模块进行打包。

# 复制不打包文件

我们需要借助一个webpack插件copy-webpack-plugin来进行拷贝我们的文件。

官网: https://www.npmjs.com/package/copy-webpack-plugin

  • 安装依赖
npm i copy-webpack-plugin
  • 引入
// webpack.config.js
const CopyPlugin = require('copy-webpack-plugin');
 
module.exports = {
  plugins: [
    new CopyPlugin([
      { from: 'source', to: 'dest' },
      { from: 'other', to: 'public' },
    ]),
  ],
};
  • 详细配置
Name Type Default Description
from (opens new window) {String\|Object} undefined Glob or path from where we сopy files.
to (opens new window) {String} compiler.options.output Output path.
context (opens new window) {String} options.context \|\| compiler.options.context A path that determines how to interpret the from path.
toType (opens new window) {String} undefined Determinate what is to option - directory, file or template.
test (opens new window) {String\|RegExp} undefined Pattern for extracting elements to be used in to templates.
force (opens new window) {Boolean} false Overwrites files already in compilation.assets (usually added by other plugins/loaders).
ignore (opens new window) {Array} [] Globs to ignore files.
flatten (opens new window) {Boolean} false Removes all directory references and only copies file names.
cache (opens new window) {Boolean\|Object} false Enable transform caching. You can use { cache: { key: 'my-cache-key' } } to invalidate the cache.
transform (opens new window) {Function} undefined Allows to modify the file contents.
transformPath (opens new window) {Function} undefined Allows to modify the writing path.

# 在模板index.html中引入

在html中引入不打包的配置文件，一般不打包文件都放在根目录下的static文件夹中，打包后放在dist文件夹。在html文件中使用script标签引用即可。注意这时候在其他组件文件中需要访问的话，可以在config.js中定义全局变量，这时候在其他文件中可以直接通过window.*进行访问。

本例中的config.js

window.URL_CONFIG = [
  {
    "url": "测试url"
  },
  {
    "url": "测试url"
  },
  {
    "url": "测试url"
  }
];

本例中的public/index.html

...
<div id="root"></div>
<script src="./static/config.js"></script>
...

在组件中访问src/pages/index/index.jsx

console.log(window.URL_CONFIG);

Bingo!
